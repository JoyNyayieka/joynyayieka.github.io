MbogaLink

Fullstack Web Application

MbogaLink revolutionizes the way you shop for fresh produce, connecting you directly with local vendors. The platform simplifies the process, allowing you to discover a wide range of fresh, locally-sourced products from the comfort of your home.

Tech Stack: Python | Bootstrap | MySQL | DarajaAPI

Key Achievements

  • The platform provides a robust product management system that allows vendors to capture and upload images of their fresh produce directly from their mobile devices while managing real-time price updates.
  • A mobile-responsive user interface was engineered using Bootstrap and JavaScript to ensure that the application remains accessible and performant for users on low-bandwidth connections and entry-level smartphones.
  • The system utilizes a secure Django authentication framework to provide distinct, protected environments for both vendors managing inventory and customers browsing local stock.
  • By applying an MVC architecture, the application effectively separates data logic from the user interface, resulting in a scalable codebase that supports rapid feature deployment.

Lessons Learned

  • Building the full-stack architecture from the ground up reinforced the importance of planning for data migrations, specifically the eventual transition from SQLite3 to a more robust production database like MySQL.

Future Improvements

  • While the current architecture successfully handles core MVP features, I am looking to refactor the project into a more modular structure. Transitioning from a monolithic design to a decoupled architecture will improve concerns like separation of logic and allow for easier integration of third-party services as the platform scales.

MovieGalore

React App

A fully functional movie search and discovery web app. The project consumes the OMDB REST API to fetch live movie data, renders results as a dynamic card grid, and supports real-time search with debouncing.

Tech Stack: React | Vite | REST API | Hooks

Key Achievements

  • Designed a scalable component architecture, emphasizing composability, separation of concerns, and reusable UI patterns.
  • Implemented reactive state-driven UI logic using React Hooks (useState, useEffect) to synchronize component state with dynamic user interactions.
  • Built data-driven interfaces through conditional rendering, array mapping, and prop-based component communication.
  • Utilized modern frontend tooling and npm-based dependency management to streamline development and optimize build performance.

Lessons Learned

  • How React state updates trigger re-renders and how improper state placement can cause unnecessary renders in parent components.
  • When to lift state up to share data between components versus keeping state local to avoid tight coupling.
  • How useEffect manages side effects, including understanding dependency arrays to prevent unnecessary executions.
  • The importance of stable keys when rendering lists to help React efficiently reconcile DOM updates.
